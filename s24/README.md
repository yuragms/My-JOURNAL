# S24 Vision

Нативное Android-приложение для **Samsung Galaxy S24**: распознаёт объекты камерой
(YOLOE-26), пишет видео с рамками, обучает/дообучает профили людей (лицо + тело) и
объектов по роликам, показывает нагрузку CPU/GPU. Всё работает **на устройстве**,
без облака и интернета.

## Что умеет

- GUI на Jetpack Compose, доступ к **обеим камерам** (селфи и основная).
- Распознавание объектов в реальном времени (YOLOE-26, prompt-free) + **отдельный детектор дронов** (YOLO26n, HuggingFace).
- Запись видео с нарисованными рамками в `Movies/S24Vision` при обнаружении объектов.
- Распознавание людей: **SCRFD** (детекция лица в кропе) → выравнивание по **5 точкам** →
  **ArcFace**-эмбеддинг; тело — **OSNet ReID**. Имя по лицу только если лицо реально
  найдено; иначе — по телу или подпись `person`.
- Обучение по ролику: вводите имя и отвечаете «это человек?»:
  - **человек** → лицо (InsightFace SCRFD + ArcFace) + тело (OSNet ReID) раздельно;
  - кадры **без видимого лица** не попадают в профиль лица;
  - **объект** → эмбеддинг кропа (MobileNetV3), бокс берётся автоматически.
- Дообучение по новому ролику: профиль есть → «улучшен: было N, стало M»;
  нет → «создан новый».
- Оверлей нагрузки: CPU % и GPU % (GPU — best-effort, может быть `n/a`).
- Экраны меню (Записи, Обучение, Профили) — **серый фон**, контрастнее со строкой уведомлений Android.

Установленное приложение на S24: **~171 MB** (из них ~76 MB — все ONNX-модели).

## Требования

- Mac M1 (или любой) с **Android Studio** (Apple Silicon) и Android SDK 35.
  Android Studio приносит JDK 17, SDK, `adb` и Gradle — отдельно их ставить не нужно.
- Samsung S24 с Android 9+.
- Кабель USB.

## 3. Установка на Samsung S24

1. На S24: **Настройки → Сведения о телефоне → Сведения о ПО** → 7 раз нажать
   «Номер сборки» → включится режим разработчика.
2. **Настройки → Параметры разработчика** → включить **Отладка по USB**.
3. Подключить S24 к Mac кабелем, на телефоне разрешить отладку с этого компьютера.
4. Проверить подключение:

```bash
adb devices    # должен появиться серийный номер устройства
```

5. Установить и запустить:

```bash
cd /Users/yuragms/Cursor-cash/s24
./gradlew :app:installDebug
adb shell am start -n com.s24vision.app/.ui.MainActivity
```

(или вручную: `adb install -r app/build/outputs/apk/debug/app-debug.apk`)

6. При первом запуске разрешите доступ к камере.

## 3b. Обновление по Wi-Fi (без кабеля)

Удобно для повторных заливок: телефон и Mac должны быть в **одной Wi-Fi-сети**.
Использует беспроводную отладку Android (Android 11+).

### Шаг 1. Включить беспроводную отладку на S24

1. **Настройки → Параметры разработчика**.
2. Включить **Беспроводная отладка (Wireless debugging)** → подтвердить.

### Шаг 2. Сопряжение (только при первом подключении)

1. На экране «Беспроводная отладка» тапнуть **«Связать устройство с помощью кода»**.
2. Появятся **IP:порт сопряжения** и **6-значный код**.
3. На Mac (порт и код живут ~1 минуту, успевайте):

```bash
adb pair <IP:порт_сопряжения> <код>
# пример: adb pair 192.168.3.233:45831 374783
```

Должно вывести `Successfully paired`.

### Шаг 3. Подключение

1. На главном экране «Беспроводная отладка» (под именем устройства) посмотреть
   **IP-адрес и порт** — он **другой**, чем порт сопряжения.
2. Подключиться и проверить:

```bash
adb connect <IP:порт_подключения>   # пример: adb connect 192.168.3.233:38457
adb devices                          # должно показать устройство SM_S9210
```

### Шаг 4. Залить новую версию

```bash
cd /Users/yuragms/Cursor-cash/s24
export JAVA_HOME="/Applications/Android Studio.app/Contents/jbr/Contents/Home"
./gradlew :app:assembleDebug
adb install -r app/build/outputs/apk/debug/app-debug.apk
adb shell am start -n com.s24vision.app/.ui.MainActivity
```

`-r` сохраняет данные/разрешения. После работы можно отключиться: `adb disconnect`.

> Сопряжение (шаг 2) нужно только один раз; далее достаточно шага 3. Если порт
> подключения изменился (после перезагрузки/переinclude отладки) — повторите `adb connect`
> с новым портом, при ошибке аутентификации повторите шаг 2.

## 4. Использование

- **Камера**: кнопка «Камера» переключает селфи/основную; рамки и подписи рисуются
  в реальном времени; сверху CPU %/GPU %; «REC» начинает запись, «Стоп» завершает.
- **Записи**: список роликов из `Movies/S24Vision` с метаданными (кадры, fps, размер),
  кнопка **▶** для просмотра, **Обучить** — переход к обучению, **del** (красная) — удалить ролик. Символ **✓** — ролик
  уже использовался для обучения, **·** — ещё нет. Новые файлы именуются коротко:
  `140626-12.30.mp4` (ддММгг-ЧЧ.мм). При первом входе запросится доступ к видео.
- **Обучение**: поля **«Имя человека / объекта»** и **«Номер бокса на видео»** (`#1`, `#2`…).
  Номер закрепляется за объектом на всём ролике (IoU-трекинг). Кнопки **Обучить** / **Дообучить**.
  Для людей: в профиль лица попадают только кадры, где SCRFD нашёл лицо (анфас/профиль).
  Во время обработки — прогресс %, CPU/GPU и имя потока.
- **Камера**: полупрозрачные боксы с номерами; подпись `имя face:X.XX body:X.XX total:X.XX`
  или `person`, если ни лицо, ни тело не прошли порог; сверху CPU/GPU и поток `camera-analysis`.
- **Профили**: список людей/объектов, удаление.

### Пороги распознавания (по умолчанию)

| Канал | Порог | Примечание |
|-------|-------|------------|
| Лицо  | 0.45  | Только если SCRFD нашёл лицо в кропе |
| Тело  | 0.65  | OSNet ReID по всему кропу человека |
| Объект| 0.50  | MobileNetV3-эмбеддинг |

Имя выставляется, если **лицо ≥ порога** или **тело ≥ порога** (логика OR).
Человек со спиной к камере не получит имя по «лицу» — только по телу или `person`.

> **После обновления с SCRFD** переобучите профили лиц (Обучить/Дообучить на ролике
> с видимым лицом): старые эмбеддинги собирались без детектора лица и несовместимы.

## 5. Где файлы на телефоне

Видео записей (`.mp4` с нарисованными рамками) сохраняются через MediaStore в
**публичную папку** общего хранилища (рядом с папками Viber/Telegram):

```
/storage/emulated/0/Movies/S24Vision/
```

Эта папка:

- видна в проводнике в общем каталоге `Movies/` и в приложении «Галерея»;
- для просмотра всех роликов в списке приложения нужно разрешение «Видео» (`READ_MEDIA_VIDEO`);
- записи, созданные приложением, пишутся через MediaStore (имя `ддММгг-ЧЧ.мм.mp4`);
- записи остаются после удаления приложения (это пользовательские медиафайлы).

Профили (эмбеддинги) лежат в приватной внутренней памяти приложения
(`/data/data/com.s24vision.app/files/profiles/`, через файловый менеджер не видны):

```
files/profiles/
├── faces/             профили лиц (<имя>.bin)
├── bodies/            профили тел
└── objects/           профили объектов
```

Скопировать записи на Mac:

```bash
adb pull /storage/emulated/0/Movies/S24Vision ./recordings
```

## 7. Известные места для доводки на устройстве

Эти участки помечены в коде и зависят от реальных артефактов (форма ONNX, железо):

- `YoloeDetector.decode()` / `CocoLabels` — формат выхода и словарь меток YOLOE
  после экспорта.
- `ScrfdDetector` / `FaceAlign` — пороги `detThresh` (0.5), `minFaceSize` (20 px),
  размер входа детектора (128/192/320) — калибровка на роликах.
- `OverlayView` — масштаб боксов под режим превью.
- `GpuMonitor` — доступность Adreno sysfs (иначе `GPU: n/a`).

## Архитектура

```
CameraX → YOLOE-26 (боксы, IoU-трекинг #N)
              │
              ├─ person? ── да → кроп человека
              │                    ├─ SCRFD (face_det) → 5 точек → ArcFace 112×112
              │                    │       └─ нет лица → face = null
              │                    └─ OSNet (body_reid) → эмбеддинг тела
              │                              │
              │                    косинус с profiles/faces + profiles/bodies
              │                              │
              │                    имя если face≥0.45 ИЛИ body≥0.65, иначе person
              │
              ├─ drone_det (YOLO26n) ── merge с YOLOE, приоритет у специалиста
              │
              └─ объект → MobileNetV3 (object_encoder) → косинус с profiles/objects
```

«Обучение» = сбор эмбеддингов из кадров видео (не тренировка весов). Распознавание
= косинусная близость к сохранённым эмбеддингам. Та же парадигма, что в desktop-
проекте `hikvision-test1`.
