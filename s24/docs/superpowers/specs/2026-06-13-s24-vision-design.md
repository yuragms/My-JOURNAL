# S24 Vision — дизайн (spec)

Дата: 2026-06-13
Статус: согласован пользователем (подход A)

## 1. Цель

Нативное Android-приложение для Samsung Galaxy S24, которое:

1. Имеет графический интерфейс и доступ к обеим камерам (селфи и основная).
2. В реальном времени распознаёт объекты (машинное зрение) через YOLO26.
3. При обнаружении объектов записывает в папку видео с нарисованными рамками.
4. По окончании трансляции (или по выбранному из папки ролику) позволяет «обучить»
   новый профиль: пользователь вводит имя и отвечает «это человек?».
   - **Человек** → распознавание по лицу (InsightFace) и телу (OSNet ReID) раздельно.
   - **Не человек** → объект: YOLOE prompt-free сам находит бокс → MobileCLIP
     эмбеддинг кропа → профиль `profiles/objects/<имя>`. В новых видео рамка с
     этим именем. До обучения такой объект показывается как «неопознанный объект».
5. Поддерживает **дообучение**: выбрать ролик + ввести имя.
   - Профиль существует → добавить эмбеддинги, сообщение «улучшен: было N, стало M».
   - Профиля нет → сообщение «нет, создаю новый» и создать.
6. Показывает нагрузку на ЦП и графическое ядро в процентах.

Целевое устройство: Samsung S24 (Snapdragon 8 Gen 3 / Adreno 750 в большинстве регионов).
Среда разработки: Mac M1.

## 2. Ключевая концепция «обучения»

«Обучение» здесь — **не** тренировка нейросети с нуля и не fine-tuning весов, а
**enrollment**: прогон готовых ONNX-моделей по кадрам видео, сбор 512-мерных
эмбеддингов и сохранение их в профиль. Распознавание = косинусная близость к
сохранённым эмбеддингам. Это та же логика, что в существующем desktop-проекте
`/Users/yuragms/Cursor-cash/hikvision-test1` (`enroll_person.py`,
`update_yura_profile.py`, `known_faces/`, `known_bodies/`).

Следствие: «обучение» — это инференс + векторная математика, поэтому выполнимо
на телефоне через ONNX Runtime без PyTorch и без встроенного Python.

## 3. Подход (выбран A)

Нативный Android (Kotlin + Jetpack Compose + CameraX) + **ONNX Runtime Mobile**
для всех моделей. Никакого Python на устройстве. Инференс и enrollment — Kotlin.

Отклонённые варианты:
- B: встроенный Python (Chaquopy) — `insightface`/`torchreid` тяжелы и капризны
  на Android, PyTorch под Android нет, APK 150+ МБ, медленнее. Нарушает KISS.
- C: гибрид Kotlin+Python — две среды исполнения, сложнее поддерживать.

## 4. Модели

Готовятся один раз на Mac скриптом-экспортёром и кладутся в
`app/src/main/assets/models/`.

| Назначение | Модель | Источник |
|---|---|---|
| Детектор (локализация всего объектного + типы) | `YOLOE-26` prompt-free размер `s` → ONNX | open-vocabulary YOLO26, встроенный словарь RAM++ (4585 классов) |
| Лицо: детекция + 5 точек выравнивания | InsightFace SCRFD (`det`) | `~/.insightface/models/buffalo_sc/` |
| Лицо: эмбеддинг (ArcFace) | InsightFace `w600k` | там же |
| Тело (ReID) | OSNet `osnet_x0_25` → ONNX | экспорт из torchreid на Mac |
| Объект: эмбеддинг кропа | MobileCLIP-S0 image encoder → ONNX | лёгкий, для узнавания обученных объектов |

Делегаты ONNX Runtime: NNAPI / XNNPACK; GPU через NNAPI где доступно. На живом
потоке YOLOE гоняется каждый N-й кадр (настройка) для удержания FPS.

### Почему prompt-free YOLOE + эмбеддинги (а не visual-prompt)

ВАЖНОЕ ОГРАНИЧЕНИЕ ONNX: в YOLOE визуальные промпты «запекаются» при экспорте, и
экспортированный ONNX **не принимает новые промпты на устройстве**
(`set_classes()`/`visual_prompts` на загруженном ONNX падают). Значит «обвёл
объект на телефоне → ONNX сразу его ищет» с замороженным ONNX невозможно.

Решение — заморожённые модели + единая парадигма «эмбеддинг + косинус»:

- **Локализацию** даёт `YOLOE-26` prompt-free (встроенный словарь 4585 классов):
  он сам находит и обводит всё объектное, включая объекты вне COCO (напр. дрон).
- **Узнавание по имени** — через эмбеддинг кропа (MobileCLIP) и косинус с профилем.

Поведение на кадре:
- **Известный класс COCO** → подпись классом.
- **`person`** → дополнительно лицо (InsightFace) + тело (OSNet), имя по косинусу.
- **Бокс совпал с обученным объектом (эмбеддинг ≥ порога)** → подпись именем профиля.
- **Объектное, но не опознано** → рамка **«неопознанный объект»** (можно обучить).

При обучении объекта рамку даёт сам детектор — **ручная обводка не нужна**.

## 5. Модули (границы ответственности)

- `camera` — CameraX, переключение селфи/основная, поток кадров.
- `detector` — YOLO26 ONNX: препроцессинг, инференс, постобработка боксов.
- `detector` — YOLOE-26 prompt-free: локализация всех объектов + их типы.
- `recognizer` — узнавание по эмбеддингам + косинус.
  - `FaceRecognizer` — SCRFD детект + 5-точечное выравнивание + ArcFace embedding.
  - `BodyRecognizer` — OSNet: ресайз кропа 256×128, нормализация, embedding.
  - `ObjectRecognizer` — MobileCLIP embedding кропа бокса + косинус с профилями.
- `recorder` — запись кадров с оверлеем в mp4 (Media3/MediaCodec).
- `enroller` — обучение/дообучение: сэмплинг кадров видео, сбор эмбеддингов,
  запись/слияние профилей, формирование сообщений «создан/улучшен».
- `profiles` — хранилище эмбеддингов и метаданных, загрузка/сохранение/удаление.
- `monitor` — CPU% (`/proc/stat`, `/proc/self/stat`), GPU% (vendor sysfs, best-effort).
- `ui` — экраны Compose, навигация, состояние.

## 6. Экраны (GUI)

1. **Камера** (главный): превью; переключатель селфи/основная; рамки объектов в
   реальном времени; имена при распознавании в стиле `имя face:.. body:.. total:..`;
   оверлей нагрузки (FPS, CPU %, GPU %, latency); кнопки REC / Стоп.
2. **Записи**: список роликов из `recordings/`, превью, открыть/удалить,
   кнопка «Обучить по этому ролику».
3. **Обучение**: поле «Имя», переключатель «Это человек?» (да → лицо+тело,
   нет → объект, бокс берётся автоматически), кнопка «Обучить / Дообучить»,
   прогресс и результат.
4. **Профили**: список профилей (людей/объектов), число эмбеддингов, удаление.

## 7. Поток детекции и записи

Кадр (каждый N-й) → `YOLOE-26` prompt-free отдаёт боксы с типами. Для каждого бокса:
- `person` → FaceRecognizer + BodyRecognizer, имя если косинус ≥ порога, иначе `person`;
- иначе → ObjectRecognizer считает эмбеддинг кропа и сравнивает с `profiles/objects/`:
  - совпал обученный объект (косинус ≥ порога) → подпись именем профиля («дрон»);
  - не совпал → подпись типом от YOLOE, а если тип не из COCO/низкая уверенность —
    **«неопознанный объект»** (порог в настройках, чтобы гасить шум).

Далее оверлей → при наличии детекции запись кадров с рамками в
`recordings/ГГГГ-ММ-ДД_ЧЧ-ММ-СС.mp4`. Пороги и веса (face/body/total) выносятся
в настройки приложения (аналог `.env` из hikvision; формула `total` — та же).

## 8. Поток обучения / дообучения

Вход: кнопка «Обучить» после трансляции **или** выбор ролика в «Записях».
Шаги: ввод имени + ответ «Это человек?».
- **Да**: по сэмплированным кадрам YOLO находит человека → FaceRecognizer собирает
  векторы лица, BodyRecognizer — векторы тела → сохранение в
  `profiles/faces/<имя>` и `profiles/bodies/<имя>`.
- **Нет (объект)**: по сэмплированным кадрам YOLOE prompt-free находит боксы
  объектов → ObjectRecognizer (MobileCLIP) считает эмбеддинги кропов → сохранение
  в `profiles/objects/<имя>`. Ручная обводка не нужна. Если в кадре несколько
  объектов, берётся самый крупный/уверенный бокс (как «1 объект на ролике»).

Логика существования профиля:
- Есть → добавить эмбеддинги (merge), сообщение «Профиль улучшен: было N, стало M».
- Нет → сообщение «Профиля нет — создаю новый» и создать.

Параметры сэмплинга по аналогии с `enroll_person.py`: `sample-every` (каждый N-й
кадр), `max-samples` (лимит эмбеддингов с одного ролика).

## 9. Хранилище

`Android/data/com.s24vision.app/files/` (доступно по USB/MTP):

```
recordings/            видео с рамками
profiles/faces/        <имя>.bin (FloatArray эмбеддингов) + meta.json
profiles/bodies/
profiles/objects/
```

Формат профиля: бинарный массив float32 (N × 512) + `meta.json`
(имя, тип, число эмбеддингов, даты, исходные ролики).

## 10. Мониторинг нагрузки

- **CPU %**: система — `/proc/stat`; приложение — `/proc/self/stat` (одна шкала 0–100%).
- **GPU %**: best-effort из vendor sysfs (Adreno: `/sys/class/kgsl/kgsl-3d0/gpubusy`
  или `gpu_busy_percentage`). Если путь недоступен (нет прав / Exynos-версия) —
  показывать `GPU: n/a`. Per-app GPU% Android без root не предоставляет.

## 11. Сборка и установка (Mac M1 → S24)

1. Один раз на Mac: запустить скрипт-экспортёр моделей (в venv от hikvision),
   он положит ONNX в `assets/models/`.
2. Android Studio (arm64) → открыть проект.
3. На S24: включить «Параметры разработчика» + «Отладка по USB».
4. Подключить кабелем → Run; либо `./gradlew assembleDebug` + `adb install`.
5. Отладка на устройстве — через `adb logcat`.

## 13. Вне рамок / риски

- Тестирование на физическом устройстве выполняет пользователь; помощь по логам.
- GPU% может быть `n/a` на части вариантов S24.
- Выравнивание лица (5 точек) и постобработка YOLO пишутся на Kotlin вручную.
- YOLOE visual-prompt НЕ используется: экспортированный ONNX не принимает новые
  промпты на устройстве. Объекты узнаются эмбеддингом кропа (MobileCLIP) + косинус.
- YOLOE-`s` prompt-free тяжелее nano — на живом потоке гоняется каждый N-й кадр.
- Узнавание объекта по эмбеддингу слабее, чем по лицу, и чувствительно к ракурсу.
- Prompt-free детекция шумит: порог уверенности гасит лишние «неопознанный объект».
- Не делаем: облако, аккаунты, синхронизацию между устройствами, обучение весов
  нейросетей (только enrollment-эмбеддинги).
